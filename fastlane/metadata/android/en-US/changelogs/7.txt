v1.1.5 - Staged Failover Host Optimization
<ul>
<li>Host failover now runs in two steps:</li>
<ul>
<li>Temporary host is selected immediately when the host disconnects</li>
<li>Temporary host runs probe rounds and finalizes host using latency + battery score</li>
</ul>
<li>Added dedicated failover scoring weights in failover_weights.dart</li>
<li>Added test coverage for failover host scoring behavior</li>
<li>General lint/alignment cleanup across controllers, screens, and tests</li>
</li>
</ul>
